# Иструкции по генерации

Файлы в директории public содержат материалы для игроков и генерируются автоматически на основе мастерских документов в корне проекта.

## Список публичных материалов

`public/intro.md` - файл пишется вручную, автоматически менять не надо.
`public/Атмосфера.md` - файл пишется вручную, автоматически менять не надо.
`public/Роли.md` - список ролей с кратким описанием, чтобы игроки могли выбрать роль и узнать общеизвестную информацию о других персонажах. Источник - Персонажи.md
`public/Контакт.md` - информация о пришельцах, получаемая при вселении инопланетянина в питомца. Источник - Пришельцы.md, Механики.md
`public/Общение_с_пришельцами.md` - правила для человека, получившего способность общаться с пришельцами. Источник - Пришельцы.md, Механики.md

Также каждому игроку генерируется персональная информация о его персонаже в директории characters, согласно нижеследующим указаниям.

## Дизайн загрузов (раздаточных материалов для игроков)

**Принцип:** давать проблему, не решение

Загруз не описывает возможные сделки и союзы. Он создаёт условия, при которых игрок сам к ним придёт.

### Структура загруза (макс. 2 страницы)
1. **Кто вы** — краткая биография и характер
2. **Чего вы хотите** — 2–3 цели по приоритету
3. **Что знаете только вы** — уникальные секреты и сведения
4. **Кто вам важен и почему** — с крючками, не просто «вы дружите»
5. **Что вас беспокоит** — угрозы вашим целям
6. **Стартовый вопрос** — одна конкретная вещь для выяснения на первом рауте

### Механизм уникального знания
Каждый игрок держит фрагмент информации, который обретает смысл только в сочетании с чужим:
- Свербеев знает: на Валемонте есть ордер из соседнего уезда
- Валемонте знает: там что-то есть, но приходившие не помнят
- Строганов-младший знает: кто-то скупает самогон в необычных количествах
