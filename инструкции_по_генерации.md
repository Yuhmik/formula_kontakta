# Иструкции по генерации

Файлы в директории public содержат материалы для игроков и генерируются автоматически на основе источников - мастерских документов в корне проекта.

- Часть источников содержат карточки – фрагменты текста между тегами CARD_BEGIN(имя_карточки, путь/) и CARD_END. Текст карточки без изменений сохраняется в файл путь/имя_карточки.md. Генерируются навыком `/generate-cards`.
- Персонажи.md содержит инструкции по генерации в начале самого этого файла.
- Другие источники содержат информацию в свободном формате.

## Список публичных материалов

- `public/intro.md` - файл пишется вручную, автоматически менять не надо.
- `public/Атмосфера.md` - файл пишется вручную, автоматически менять не надо.
- `public/Роли.md` - список ролей с кратким описанием, чтобы игроки могли выбрать роль и узнать общеизвестную информацию о других персонажах. Инструкцию по генерации см. ниже.
- `public/Контакт.md` - информация о пришельцах, получаемая при вселении инопланетянина в питомца. Источник - `Пришельцы.md`
- `public/Общение_с_пришельцами.md` - правила для человека, получившего способность общаться с пришельцами. Источник - `Пришельцы.md`

## Генерация публичного списка ролей `public/Роли.md`

- Основной источник - `Персонажи.md`, блоки обозначенные тегом `PUB_INFO` / `PUB_END` и идущая перед ним информация (имя, роль, возраст) – с учётом полей `пол` и `имя` из `Игроки.md`
- Каждая строка блока `PUB_INFO` становится отдельным абзацем (отделяется пустой строкой)
- добавляется портрет персонажа из `материалы\графика\персонажи\портреты.md` – блок с маркером, соответствующим полю `пол` в `Игроки.md`
- добавляется контакт и статус заявки игрока из `Игроки.md`
- в конце файла добавляется раздел примечаний из блока `FOOTER_INFO` / `FOOTER_END` файла `Персонажи.md`
- весь текст приводится к нужному гендеру, согласно инструкции "Смена гендера персонажа" приведённой ниже

## Игротехнические карточки
Генерируются из блоков, оборзначенных тегами `CARD_BEGIN` / `CARD_END`.

- `public/Контакт.md` – базовая информация о пришельцах для хозяина животного-носителя. Выдаётся с почтой в начале раута. Источник – `Пришельцы.md`
- `public/Общение_с_пришельцами.md` – правила для человека, получившего способность общаться с пришельцами. Выдаётся игротехником в локации "место крушения". Источник – `Пришельцы.md`
- `public/Медицина.md` – правила лечения для Пирогова и Фишнер. Источник – `Механики.md`
- `public/игротехам/Мы_пришельцы.md` – погружение для питомцев-инопланетян, впервые попадающих в локацию. Используется игротехником. Источник – `Пришельцы.md`
- `public/игротехам/Земное_Существо.md` – объяснение пришельцев людям-контактёрам. Используется игротехником. Источник – `Пришельцы.md`
- `public/игротехам/Пропуск_в_локацию.md` – инструкция по пропуску игроков на место крушения. Источник – `Пришельцы.md`
- `public/игротехам/Уровни_контакта.md` – памятка по уровням контакта. Источник – `Пришельцы.md`

## Также каждому игроку генерируется персональная информация о его персонаже в директории characters, согласно нижеследующим указаниям:

### Дизайн загрузов (раздаточных материалов для игроков)

**Принцип:** давать проблему, не решение
Загруз не описывает возможные сделки и союзы, не спойлерит сюжетные повороты. Он создаёт условия, при которых игрок сам к ним придёт.

### Структура загруза (макс. 2 страницы)
1. **Кто вы** — краткая биография и характер
2. **Чего вы хотите** — 2–3 цели по приоритету
3. **Что знаете только вы** — секреты и уникальные сведения
4. **Кто вам важен и почему** — с крючками, не просто «вы дружите»
5. **Что вас беспокоит** — угрозы вашим целям
6. **Стартовый вопрос** — одна конкретная вещь для выяснения на первом рауте

## Смена гендера персонажа

При генерации любых публичных материалов – карточки игрока, публичного списка ролей, писем, игротехнических карточек – используй поля `пол` и `имя` из таблицы `Игроки.md`. Они задают гендер и полное имя персонажа для данного прогона игры; исходные файлы (`Персонажи.md`, `Механики.md`, `Пришельцы.md`) не изменяются.

Если поле `пол` совпадает с гендером исходного текста – материал воспроизводится без изменений. Если гендер инвертирован – применяются правила ниже.

### Грамматические замены

**Местоимения**

| М | Ж |
|---|---|
| он | она |
| его | её |
| ему | ей |
| им | ей |
| (о/в/на) нём | ней |

**Глаголы прошедшего времени:** окончание `-л` → `-ла` (приехал → приехала, знал → знала, был → была).

**Краткие прилагательные и причастия:** рад → рада, готов → готова, должен → должна, известен → известна, намерен → намерена, рождён → рождена, ограничен → ограничена.

**Родственные отношения** (когда речь идёт о самом персонаже)

| М | Ж |
|---|---|
| сын | дочь |
| брат | сестра |
| племянник | племянница |
| дядя | тётя |
| дед | бабка |
| вдовец | вдова |
| жених | невеста |
| холостяк | незамужняя |

**Обращения и роли** – указываются в описании персонажа в `Персонажи.md` (пометка «Гендер персонажа можно инвертировать» содержит нужные формы). Общие замены: господин → госпожа, барин → барыня, помещик → помещица.

**Имя** персонажа берётся из поля `имя` в `Игроки.md`.

### Где одной грамматики недостаточно

В России 1835 года положение мужчин и женщин одного сословия существенно различалось. При инверсии гендера проверяй следующие места в тексте и при необходимости адаптируй смысл, а не только форму:

**Имущество и деньги.** Замужняя женщина не распоряжается имением и финансами самостоятельно – всё юридически принадлежит мужу. Вдова или незамужняя наследница имеет бо́льшую свободу, но также ограниченную. Если персонаж владеет имением, ведёт дела или принимает финансовые решения – уточни, каков его брачный статус после инверсии, и скорректируй формулировки.

**Свобода передвижения.** Одиночные ночные поездки, визиты без сопровождения, неформальные встречи с мужчинами – для женщины всё это социальный риск или невозможность. Поручения такого рода требуют переосмысления или объяснения обстоятельств.

**Служебные и карьерные цели.** Чин, орден, служебное продвижение – недоступны женщине. Заменяй на достижимые цели: репутация, выгодный брак для себя или близких, влияние через общество или через мужа.

**Авторитет в прямом давлении.** Мужчина давит статусом или силой напрямую. Женщина, как правило, влияет через обаяние, слухи, альянсы, управление информацией. Если персонаж использует прямое принуждение – обдумай, реалистична ли такая форма давления для женщины в этом кругу.

**Брачный статус.** Незамужняя женщина находится под опекой отца или старшего родственника; замужняя – в сфере влияния мужа. Это меняет, кто формально принимает решения в её пользу и кто может её ограничить.

**Гувернантка и сопровождение.** Молодая незамужняя женщина нуждается в гувернантке или компаньонке – для приличий и как постоянная тень. Молодой человек в ней не нуждается: опекун или старший родственник отвечает за него в целом, без постоянного присутствия. При инверсии гендера такого персонажа убирай гувернантку из его истории, а ответственность за него перекладывай напрямую на опекуна. При обратной инверсии (мужчина → женщина) – добавляй фигуру гувернантки или компаньонки и учитывай её присутствие в ситуациях, где персонаж действует «в обществе».
